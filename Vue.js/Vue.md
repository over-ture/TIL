## 1. Frontend Development (프론트엔드 개발)

### 프론트엔드 개발이란?
*   웹사이트와 웹 애플리케이션의 **사용자 인터페이스(UI)와 사용자 경험(UX)**을 만들고 디자인하는 것.
*   HTML, CSS, JavaScript 등을 활용하여 **사용자가 직접 상호작용하는 부분**을 개발합니다.
*   **구조:** 클라이언트(사용자) <-> **Frontend Framework(Vue, React 등)** <-> 서버(Django 등)

## 2. Client-side frameworks (클라이언트 사이드 프레임워크)

### Client-side frameworks 란?
*   클라이언트 측에서 UI와 상호작용을 개발하기 위해 사용되는 **JavaScript 기반 프레임워크**입니다.
*   웹사이트의 UI를 효율적으로 만들기 위해 미리 짜놓은 코드의 뼈대입니다.
*   복잡한 웹 애플리케이션을 마치 **레고를 조립하듯**, 가능한 부품 단위로 쉽게 개발할 수 있게 도와줍니다.
*   오늘날 대부분의 동적인 웹사이트는 이 프레임워크를 사용하며, 대표적으로 **Vue, React, Angular**가 있습니다.

### 왜 Client-side frameworks가 필요한가?

**① 단순히 무언가를 '읽는 곳'에서 -> 무언가를 '하는 곳'으로 변화**
*   현대 사용자는 웹에서 문서만 읽는 것이 아니라, 음악 스트리밍, 영화 시청, 실시간 텍스트/영상 채팅 등을 수행합니다.
*   이러한 현대적이고 복잡한 대화형 웹 사이트를 **"웹 애플리케이션(web applications)"**이라 부릅니다.
*   프레임워크의 등장으로 매우 동적인 대화형 애플리케이션을 훨씬 쉽게 구축할 수 있게 되었습니다.

**② 다루는 데이터가 많아짐**
*   예시: 친구가 이름을 변경했다면? 친구 목록, 타임라인, 스토리 등 이름이 출력되는 모든 곳이 함께 변경되어야 합니다.
*   애플리케이션의 기본 데이터를 안정적으로 추적하고 업데이트(렌더링, 추가, 삭제 등) 하는 도구가 필요합니다.
*   상태가 변경될 때마다 UI가 일관성 있게 업데이트되어야 합니다.

**③ Vanilla JS만으로는 간단하지 않음**
*   순수 자바스크립트(Vanilla JS)만 사용하면, 데이터가 바뀔 때마다 모든 DOM 요소를 일일이 찾아 직접 수정(`textContent = ...`)해야 하므로 **불필요한 코드의 반복**이 발생하고 관리가 어렵습니다.

### 💡 Vue.js의 해결책: '자동화'와 '반응성'
> "데이터가 바뀔 때마다 매번 DOM 요소를 찾아 수정하는 작업, 더 편하게 할 수는 없을까요?"

*   Vue는 이 DOM 수정 과정을 **완전히 자동화**합니다.
*   개발자가 할 일은 **'데이터가 바뀌면 UI는 이런 모습이어야 한다'고 선언**하는 것뿐입니다. (선언적 렌더링)
*   데이터가 바뀌면 화면이 '알아서' 바뀌는 Vue의 **반응성(Reactivity)** 덕분에, 우리는 더 이상 번거로운 DOM 조작에 신경 쓸 필요가 없습니다.

### 요약: Client-side frameworks의 필요성 3가지
1.  **동적이고 반응적인 웹 애플리케이션 개발:** 실시간 데이터 업데이트 용이
2.  **코드 재사용성 증가:** 컴포넌트 기반 아키텍처 및 모듈화된 코드 구조
3.  **개발 생산성 향상:** 강력한 개발 도구 지원

## 3. SPA (Single Page Application)

### SPA란?
*   **단일 페이지**에서 동작하는 웹 애플리케이션입니다.
*   하나의 HTML 파일 위에서 JavaScript가 **필요한 부분만 교체**하며, '진짜' 페이지 이동(새로고침) 없이 동작합니다.
*   마치 하나의 무대에서 배우와 배경만 계속 바꾸는 연극과 같습니다.
*   앱처럼 빠르고 부드러운 사용자 경험(UX)을 제공합니다.

### SPA의 작동 원리 및 CSR
1.  **최초 로드 시:** 애플리케이션에 필요한 주요 리소스를 한 번에 다운로드합니다.
2.  **페이지 갱신 시:** 필요한 데이터만을 비동기적으로 전달받아 화면의 **필요한 부분만 동적으로 갱신**합니다.
    *   AJAX와 같은 기술을 사용하여 필요한 데이터만 서버로부터 가져옵니다.
    *   페이지 전체를 다시 로드할 필요가 없습니다.
3.  JavaScript를 사용하여 **클라이언트 측에서 동적으로 콘텐츠를 생성하고 업데이트**합니다.
    *   👉 이러한 렌더링 방식을 **CSR (Client-Side Rendering)** 방식이라고 합니다.


## 4. CSR (Client-Side Rendering)

### Client Side Rendering 이란?
*   **클라이언트(브라우저)에서 콘텐츠를 렌더링하는 방식**입니다.
*   **비유:** 일단 빈 집(HTML)에 들어간 뒤, 가구(JavaScript)를 배송받아 직접 조립하는 방식과 같습니다.
*   브라우저는 서버로부터 거의 텅 빈 HTML과 JavaScript 파일만을 받아옵니다. 그 후 JavaScript가 실행되어 필요한 데이터를 요청하고 화면을 동적으로 완성합니다.

### CSR 작동 원리
1.  사용자가 웹사이트에 요청을 보냅니다.
2.  서버는 **최소한의 HTML과 JavaScript 파일**을 클라이언트로 전송합니다.
3.  클라이언트는 HTML과 JavaScript를 다운로드합니다.
4.  브라우저가 **JavaScript를 실행하여 동적으로 페이지 콘텐츠를 생성**합니다.
5.  필요한 데이터는 API를 통해 서버로부터 **비동기적**으로 가져옵니다.
    *   *💡 비동기: 하나의 작업이 끝날 때까지 기다리지 않고 다른 작업을 실행하는 방식*
    *   *💡 DOM: 자바스크립트가 HTML을 조작할 수 있게 만든 객체 모델*

### CSR 작동 예시 (Google Maps, Facebook, Instagram 등)
*   초기 요청 시 뼈대만 있는 HTML과 앱 실행에 필요한 JS를 받습니다.
*   이후부터는 서버가 더 이상 HTML을 제공하지 않고, **요청에 필요한 데이터(JSON)만 응답(Ajax 통신)**합니다.
*   클라이언트 측에서 JS를 사용해 DOM을 업데이트하므로 **새로고침 없이** 페이지가 갱신됩니다.

## 5. CSR과 SPA의 장단점

### 👍 장점
1.  **빠른 페이지 전환**
    *   초기 로드 후에는 필요한 데이터만 가져와 페이지 일부만 다시 렌더링하므로 속도가 빠릅니다.
    *   서버로 전송되는 데이터 양이 최소화되어 서버 부하를 방지합니다.
2.  **향상된 사용자 경험 (UX)**
    *   페이지 새로고침이 발생하지 않아 네이티브 앱과 유사한 부드러운 사용자 경험을 제공합니다.
3.  **Frontend와 Backend의 명확한 분리**
    *   **FE:** UI 렌더링 및 사용자 상호 작용 처리 담당
    *   **BE:** 데이터 및 API 제공 담당
    *   대규모 애플리케이션 개발 및 유지 관리가 훨씬 쉬워집니다.

### 👎 단점
1.  **느린 초기 로드 속도**
    *   JS가 다운로드, 구문 분석 및 실행될 때까지 빈 화면이 보일 수 있어 약간의 지연이 발생합니다.
2.  **SEO (검색 엔진 최적화) 문제**
    *   검색 엔진은 HTML을 읽어서 분석하는데, 초기 HTML이 비어있어 검색에 잘 노출되지 않을 수 있습니다.

> **💡 TIP:** 초기 로딩 속도와 SEO 문제는 존재하지만, 로딩 후의 압도적으로 훌륭한 UX가 단점을 상쇄합니다. 최근에는 **Next.js, Nuxt.js 같은 하이브리드 프레임워크(SSR 지원)**를 통해 이러한 단점을 대부분 해결하면서 SPA의 장점을 그대로 누릴 수 있습니다.

## 6. Rendering 방식 비교 요약

| 방식 | 설명 | 특징 |
| :--- | :--- | :--- |
| **MPA** | Multi Page Application | 사용자가 다른 페이지로 이동할 때마다 서버로부터 **새로운 HTML 파일을 매번 로드**함 |
| **SPA** | Single Page Application | 단일 HTML에서 JS를 이용해 동적으로 화면을 교체함 |
| **SSR** | Server-side Rendering | **서버에서 화면을 렌더링**하는 방식. 모든 데이터가 담긴 완성된 HTML을 클라이언트에 전달함 |
| **CSR** | Client-side Rendering | **클라이언트(브라우저)에서 화면을 렌더링**하는 방식. |

## 7. Vue.js 소개

### Vue 란?
*   사용자 인터페이스(UI)를 구축하기 위한 **JavaScript 프레임워크**입니다.
*   레고 블록처럼 재사용 가능한 부품(**컴포넌트**)으로 화면을 조립합니다.
*   데이터가 바뀌면 화면도 자동으로 바뀌는 **'반응성'**이 가장 큰 특징입니다.
*   2014년 Evan You(전 Angular 개발팀)에 의해 발표되었으며, 현재 **최신 버전은 "Vue 3"** 입니다. *(학습 시 Vue 2 문서를 보지 않도록 주의!)*

### Vue의 주요 특징
1.  **반응형 데이터 바인딩:** 데이터 변경 시 UI가 자동으로 업데이트됩니다.
2.  **컴포넌트 기반 아키텍처:** 재사용 가능한 UI 조각으로 화면을 구성합니다.
3.  **간결한 문법과 직관적인 API:** 가독성이 높고 학습 곡선이 낮습니다.
4.  **유연한 스케일링:** 작은 토이 프로젝트부터 대규모 애플리케이션까지 모두 적합합니다.

### Vue를 학습하는 이유 (SSAFY 기준 포함)
*   React나 Angular에 비해 **문법이 간결하고 직관적**이라 짧은 시간 내에 효율적으로 결과물을 만들 수 있습니다.
*   잘 정리된 공식 문서와 활성화된 글로벌 커뮤니티 덕분에 풍부한 리소스와 예제를 얻기 쉽습니다.
*   다양한 플러그인과 라이브러리를 제공하여 **확장성**이 매우 높습니다.

## 8. Vue의 2가지 핵심 기능

### 1. 선언적 렌더링 (Declarative Rendering)
*   표준 HTML을 확장하는 Vue의 **"템플릿 구문(`{{ }}` 등)"**을 사용합니다.
*   JavaScript 상태(데이터)를 기반으로 화면에 출력될 HTML을 **선언적**으로 작성합니다.
*   *"이 데이터가 여기에 보여야 해"* 라고 선언만 하면 Vue가 알아서 그려줍니다.

### 2. 반응성 (Reactivity)
*   JavaScript **상태(데이터)의 변경을 추적**합니다.
*   데이터에 변경사항이 발생하면 **자동으로 DOM을 업데이트**하여 화면을 바꿉니다.

## 9. Vue 코드 체험하기 (Basic Example)

아래는 CDN을 활용하여 Vue 3의 핵심 기능을 구현한 간단한 예시입니다.

```html
<!-- HTML 부분 (View) -->
<div id="app">
  <!-- 1. 선언적 렌더링: message 데이터가 바인딩 됨 -->
  <h1>{{ message }}</h1>
  
  <!-- 2. 이벤트 처리 및 반응성: 클릭 시 count가 증가하고 화면이 자동 갱신됨 -->
  <button v-on:click="count++">
    Count is: {{ count }}
  </button>
</div>

<!-- JavaScript 부분 (Logic) -->
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script>
  const { createApp, ref } = Vue

  const app = createApp({
    setup() {
      // ref()를 사용하여 '반응형 데이터' 생성
      const message = ref('Hello vue!')
      const count = ref(0)

      // 템플릿(HTML)에서 사용할 데이터를 반환(return)
      return {
        message,
        count
      }
    }
  })

  // id가 'app'인 요소에 Vue 앱을 연결(mount)
  app.mount('#app')
</script>
```
