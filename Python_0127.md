# Data Structure 2

## 딕셔너리

**Dictionary**

- 키(Key)와 값(Value)를 짝지어 저장하는 자료구조
- 내부적으로 해시 테이블을 사용하여 키-값 쌍을 관리
- 키를 통한 삽입, 삭제, 검색이 데이터의 크기와 관계없이 매우 빠르다

---

### 딕셔너리 메서드

**`.get(key[,default])`**

- 키와 연결된 값을 반환하거나 키가 없으면 None 혹은 기본값 반환

**`.keys()`**

- 딕셔너리 키를 모은 객체를 반환
  

**`.values()`**

- 딕셔너리 값를 모은 객체를 반환
  
**`.items()`**

- 딕셔너리 키/값 쌍을 모은 객체를 반환
- 

**`.pop(key[,default])`**

- 키를 제거하고 연결됐던 값을 반환 (없으면 에러나, default를 반환)
  
**`.clear()`**

- 딕셔너리의 모든 키/값 쌍을 제거

**`.setdefault(key[,default])`**

- 키와 연결된 값을 반환
- 키가 없다면 `default`와 연결한 키를 딕셔너리에 추가하고 `default`를 반환

---

### 세트 메서드

**`.add(x)`**

- 세트에 `x`를 추가
  

**`.update(iterable)`**

- 세트에 다른 `iterable` 요소를 추가
  

**`.clear()`**

- 세트의 모든 항목을 제거

**`.remove(x)`**

- 세트에서 항목 `x`를 제거, 항목 `x`가 없을 경우 `keyError`

**`.pop()`**

- 세트에서 **임의의** 요소를 제거하고 **반환**
  

**`.discard(x)`**

- 세트에서 항목 `x`를 제거. `remove`와 달리 에러 없음

## 딕셔너리 확장: defaultdict

### defaultdict

- 파이썬 내장 모듈 `collections`에서 제공하는 딕셔너리의 확장판
- 딕셔너리에 존재하지 않는 키를 조회할 때, 자동으로 '기본값'을 생성해주는 자료구조
- `KeyError` 걱정 없이 값을 수정하거나 추가할 때 매우 유용함
  
#### defaultdict 구문 및 활용 가이드
```python
defaultdict(자료형)
```
- 사용법: defaultdict(자료형) 형태로 선언

- 자주 쓰는 패턴
  1. 숫자 세기: `defaultdict(int)`
       - 키가 없으면 0으로 초기화
  2. 그룹핑/리스트 모으기: `defaultdict(list)`
       - 키가 없으면 빈 리스트 []로 초기화

**defaultdict 주의사항**

- `defaultdict`는 조회만 해도 키가 생성됨
  - 예: `print(my_dict['missing_key])`를 하면 0이 출력되면서 키가 추가됨

- 따라서 '단순히 키가 있는지 확인할 때'는 주의해서 사용해야 함

## 해시 테이블

### 해시 테이블

- 해시 테이블은 키(Key)와 값(Value)을 짝지어 저장하는 자료구조

  - 책 제목(키)를 색인(해시 함수)에서 찾아 몇 번째 책장(인덱스)에 있는 지 알아내면, 해당 책장(배열)로 바로 가서 책(값)을 꺼낼 수 있다

#### 해시

- 임의의 크기를 가진 데이터를 고정된 크기의 **고유한 값**으로 변환하는 것

  - 생성된 해시 값(고유한 정수)은 해당 데이터를 식별하는 '지문'역할을 함
  - 파이썬에서는 이 해시 값을 이용해 해시 테이블에 데이터를 저장
  - 이 변환을 수행하는 것이 **해시 함수**



## 파이썬 문법 규격

### BNF (Backus-Naur Form)

**프로그래밍 언어의 문법을 표현하기 위한 표기법**

### EBNF (Extended Backus-Naur Form)

**BNF를 확장한 표기법, 메타 기호를 추가하여 더 간결하고 표현력이 강해진 형태**

#### 대표적인 EBNF 메타 기호

| 메타 기호 | 의미 |
| :---: | :---: |
| [ ] | 선택적 요소 |
| { } | 0번 이상 반복 |
| ( ) | 그룹화 |
